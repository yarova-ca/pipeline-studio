How industry selection changes the pipeline:
Framework catalog — industry does NOT restrict frameworks. Any framework runs in any industry.
Exception: FIPS-140-2 compliance affects some runtimes. Rows with this restriction carry a FIPS required note in Framework notes.
Pipeline tool catalog — industry DOES change the pipeline. Industry → auto-selects compliance standard → adds extra controls at specific stages.
Canadian version (73 industries, provincial regulators): 07-canada-schema.html
Column definitions — all 10

1 Industry — name of the industry.

2 Vertical — parent group (one of 16 verticals).

3 Mandatory compliance — legally required standards for this industry.

4 Optional compliance — best-practice standards, not legally required.

5 Regulatory body — the authority that enforces mandatory compliance.

6 Data sensitivity — type of sensitive data handled: PII / PHI / PCI / Classified / Public.

7 Pipeline stages affected — the specific stages that get extra controls when this industry is selected.

8 Framework notes — any compliance-driven framework restrictions. "None" = all frameworks permitted.

9 Key security requirements — top security requirements specific to this industry.

10 Audit requirements — log retention period and audit trail requirements.

Catalog connections

Framework catalog — industry has no effect

Any framework runs in any industry.
Industry does NOT restrict framework choice.
Exception: FIPS-140-2 required industries restrict runtimes to FIPS-certified builds.

Pipeline tool catalog — industry changes the pipeline

Industry → auto-selects compliance standard.
Compliance → adds extra controls at specific stages.
Those stages generate different files with those controls included.
