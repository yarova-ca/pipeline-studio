Notes on * partial cells

Groups 01, 04, 05, 07 — COMPLIANCE and OBSERVABILITY marked *

COMPLIANCE applies at the HTTP handler level — same pattern as pure API servers. Y behavior.

OBSERVABILITY: OTel SDK applies at server startup — same pattern. Y behavior.

Marked * as a reminder that these are full-stack frameworks — verify the HTTP handler entry point before adding middleware.

Groups 01, 04, 05, 07 — AUTH marked *

What works: JWT Bearer middleware on API routes within the framework.

What does not work: Standard JWT middleware on page routes (SSR pages use cookies / framework-native auth).

Next.js: Use Next-Auth for page auth. JWT Bearer middleware for /api/* routes only.

SvelteKit: Use SvelteKit hooks (hooks.server.ts) for session. JWT Bearer for API endpoints.

Remix: Use Remix sessions for page auth. JWT Bearer for resource routes.

Group 06 — PKG_MANAGER marked *

Group 06 (Cloudflare Workers) has no Dockerfile — RUNTIME and BUILD_TOOL do not apply.

PKG_MANAGER applies to the local lock file used by wrangler deploy, not via a docker build --build-arg.

Set the package manager in the project's package.json packageManager field — not via Docker.
