What this doc covers
The tool selected for every pipeline stage. What it does, when to use it, and the closest alternative.
Structure: Phase → stages within it → one tool table per stage.
Stage classification (DevOps vs DevSecOps): doc 03. Ordering constraints: doc 05. Pipeline flow: doc 02.
Column definitions — all 10 columns explained
ToolName + current stable version. Verify against the GitHub releases page before pinning in CI.
LicenseOpen source license or commercial model. Affects procurement and legal review requirements.
Applies toAll stacks / specific language / container-only / Kubernetes-only. Determines when to skip this tool.
Changes whenThe condition that requires reconfiguring this tool — not how to configure it, just the trigger.
Output formatFormat of findings. SARIF uploads to GitHub Security tab. JSON is consumed by downstream tools.
CI integrationHow the tool runs in CI. Native action = no Docker pull. Docker image = slower cold start. CLI = must be installed.
MandatoryYes = skip creates a security hole. Conditional = mandatory only for specific stack types. No = optional improvement.
When to useThe specific condition under which this tool is the right choice. No vague statements.
When NOTThe specific condition to skip or replace this tool. Names the alternative.
Primary tradeoffvs the closest real alternative — what you gain, what you give up. One line.
