Column legend

ColumnWhat it means
HLTHHealth endpoints — /health, /health/live, /health/ready — all three present and returning correct HTTP status codes.
DOCKDockerfile — multi-stage builder + runtime image. Not applicable for mobile and edge services that deploy outside containers.
HCHKHEALTHCHECK instruction inside the Dockerfile — tells the container runtime how to detect a stuck process. Not applicable where DOCK is N/A.
DIGN.dockerignore — keeps build context small, prevents secrets from leaking into the image layer cache.
CIGitHub Actions workflow — all 6 phases: test, scan, build, push, deploy, notify.
HELMHelm chart — Kubernetes deployment with dev / staging / prod value files. Not applicable for mobile and edge services.
KUSTKustomize overlays — GitOps alternative to Helm for the same Kubernetes deployment. Not applicable for mobile and edge services.
AUTHAuthentication — JWT + API key for backend APIs, OAuth2 + PKCE for frontend services.
ORMDatabase layer — Prisma / SQLAlchemy / GORM / equivalent — models, migrations, and typed query helpers. Not applicable for pure frontend / SPA services.
LOGStructured JSON logging — pino / structlog / slog / equivalent — machine-readable output with request ID and trace fields.
METPrometheus metrics — /metrics endpoint exposing RED metrics (rate, errors, duration) for Grafana dashboards.
CATBackstage catalog — catalog-info.yaml registered in the service registry so the service appears in Backstage TechDocs.
COMPCompliance configs — FIPS / HIPAA / PIPEDA / PCI / FedRAMP — Dockerfile build arg + runtime flag that switches the image to a compliant base.
