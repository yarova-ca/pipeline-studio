What an invariant is: a condition that is always true at the named stage. No exception. No partial pass.
If violated — what breaks downstream when this condition is not met. Enforced by — the specific tool or mechanism that makes the condition hold. Control class — category of the invariant:   Security — prevents an attack vector or credential exposure.   Compliance — required for supply chain attestation or audit trail.   Process — enforces workflow order or human review gate.
When adding a new stage or changing stage order: check every invariant in the affected phase. A single ordering change can break multiple downstream invariants.
Acronym definitions — terms used in this doc

SCA — Software Composition Analysis. Scans third-party dependencies for known CVEs.

SAST — Static Application Security Testing. Scans source code for insecure patterns without running the code.

DAST — Dynamic Application Security Testing. Runs the application and probes it for vulnerabilities over HTTP.

IaC — Infrastructure as Code. Terraform, Helm, Kubernetes manifests, CloudFormation, etc.

SBOM — Software Bill of Materials. Machine-readable list of every component in the build artifact.

SLSA — Supply-chain Levels for Software Artifacts. Provenance attestation proving the artifact was built by a trusted pipeline from a known source.

OIDC — OpenID Connect. Used for keyless CI authentication — each CI job gets a short-lived token, no stored credentials.

Column definitions — all 6 columns

ID — unique identifier. Format: I-<phase>-<n>. Example: I-P3-1 = Phase 3, first invariant.

Stage — the pipeline stage where this invariant is enforced.

Invariant — the exact condition that must always hold. Written as a positive statement.

If violated — what breaks downstream. Downstream = anything after this stage in the same pipeline run or in a later environment.

Enforced by — the tool, CI mechanism, or policy that enforces the invariant. If the enforcer fails or is misconfigured, the invariant is not guaranteed.

Control class — Security / Compliance / Process. See legend above for definitions.
