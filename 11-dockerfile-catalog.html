Multi-stage pattern — two FROMs, why and what gets copied

Why two stages

The build stage has compilers, package managers, build tools, and source code. The runtime stage has only the compiled output. The final image is the runtime stage — build tools never reach production.

What gets copied

Only the artifacts the app needs at runtime: compiled binary, vendor directory, or framework output directory. Source code, node_modules dev dependencies, and build toolchains are discarded.

Named targets

--target runtime builds the standard production image. --target runtime-fips builds the FIPS-compliant variant using a FIPS-validated base image (UBI9, Azure Linux FIPS, or dotnet FIPS).

ARG — package manager selection (npm vs yarn vs pnpm, same Dockerfile)

Default

All Node.js Dockerfiles default to ARG PM=npm. The install and build RUN commands check $PM and call the correct tool.

Override at build time

docker build --build-arg PM=pnpm --target runtime .

docker build --build-arg PM=yarn --target runtime .

What the ARG controls

The package manager install command only. Build and start commands are always the same (npm run build / pnpm run build / yarn run build are equivalent).

Python: ARG PM=pip

Python Dockerfiles use ARG PM=pip. Options: pip (default), poetry, uv.

Java: ARG BUILD_TOOL=maven

JVM Dockerfiles use ARG BUILD_TOOL=maven. Options: maven (default), gradle.

Named targets — --target runtime vs --target runtime-fips

Standard runtime

docker build --target runtime . — builds the standard production image. Node.js: node:22-alpine. Go/Rust: scratch. Python: python:3.12-slim. Java: distroless/java21. .NET: dotnet/aspnet:9.0-alpine.

FIPS runtime

docker build --target runtime-fips . — builds with a FIPS-validated base image. Node.js: ubi9/nodejs-22-minimal. Go/Rust: ubi9/ubi-micro. Python: ubi9/python-311. Java: ubi9/openjdk-21-runtime. .NET: dotnet/aspnet:9.0-cbl-mariner2.0-fips.

When to use FIPS

FedRAMP Moderate+, DoD IL2+, ITAR, and HIPAA environments that require FIPS 140-2 or 140-3 validated cryptographic modules. Required when the system processes classified or controlled unclassified information.

CA certificates — scratch and distroless/static have none

Images with no CA cert bundle

scratch and gcr.io/distroless/static-debian12 ship with no CA cert bundle. Go and Rust Dockerfiles in this catalog all include COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/ to fix this.

What fails without CA certs

Any outbound HTTPS call: API clients, package manager installs at runtime, registry calls. The error appears as SSL handshake failure or certificate error — not a clear "no CA certs" message.

Go and Rust Dockerfiles

The CA cert COPY is already included in all Go and Rust Dockerfiles in this catalog. Do not remove it unless your app makes zero outbound HTTPS calls.

FIPS compliance variant — which distros, when to use runtime-fips

FIPS 140-2 / 140-3

Federal Information Processing Standard. US government requirement for cryptographic modules. Required for FedRAMP Moderate+, DoD IL2+, ITAR.

FIPS-validated images in this catalog

ubi9/nodejs-22-minimal — Node.js on RHEL 9 with FIPS-validated OpenSSL.

ubi9/ubi-micro — minimal RHEL 9 runtime for Go and Rust binaries.

ubi9/python-311 — Python 3.11 on RHEL 9 with FIPS-validated OpenSSL.

ubi9/openjdk-21-runtime — JDK 21 on RHEL 9 for JVM workloads.

dotnet/aspnet:9.0-cbl-mariner2.0-fips — .NET 9 runtime on CBL-Mariner 2.0 with FIPS.

Build command

docker build --target runtime-fips -t myapp:fips .

CI-only frameworks — why no Dockerfile, what to use instead

Why no Dockerfile

iOS, Android, React Native, Flutter, .NET MAUI, and Kotlin Multiplatform apps do not run as OCI containers. The runtime is a device (iOS/Android) or a CDN edge network (Cloudflare Workers, Vercel Edge). A Dockerfile would be wrong — there is no container to run.

What CI-only means

The CI pipeline builds the native artifact (APK, IPA, AAB, Worker bundle). The artifact is deployed directly to an app store, a device fleet, or a CDN — not to a container registry.

iOS requirement

iOS builds require macos-latest GitHub Actions runner. Xcode is macOS-only. ubuntu-latest will fail immediately.

Edge rendering

Next.js Edge, Hono Cloudflare, and Remix Cloudflare deploy via wrangler deploy or vercel deploy. The config file is wrangler.toml, not a Dockerfile.

SSR / Hybrid

Server-side rendering: Node.js serves HTML on every request. Use when SEO, first-load performance, or auth-gated pages are priorities.
